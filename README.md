## 🛠 Технологический стек

*   **Бэкенд:** Python 3.12 / Django 5.x / Django REST Framework (DRF)
*   **Асинхронный сервер & Сокеты:** Django Channels (ASGI) / Daphne
*   **Очереди & Фоновые задачи:** Celery / RabbitMQ (брокер тасок)
*   **Кэш & Channel Layers:** Redis
*   **ИИ-Интеграция:** OpenAI API 
*   **Уведомления:** Aiogram (Telegram Bot API)
*   **База данных:** PostgreSQL 17
*   **Веб-сервер / Прокси:** Nginx
*   **Тестирование:** Pytest-Django
*   **Окружение:** Docker / Docker Compose

## ⚡️ Ключевые особенности и архитектура

1.  **WebSockets (Протокол 101):** Полный отказ от HTTP Long-Polling в пользу постоянных двухсторонних WebSocket-соединений через Daphne. Сообщения, статусы и системные события доставляются мгновенно.
2.  **Асинхронная обработка БД:** Все операции сохранения сообщений и проверки прав доступа пользователей внутри `ChatConsumer` обёрнуты в асинхронный контекст `database_sync_to_async`.
3.  **Фоновые задачи и Нотификации:** Интегрирована связка **Celery + RabbitMQ** для выноса «тяжелых» процессов (отправка email, обработка подписок) из основного потока приложения.
4.  **Telegram Bot Экосистема:** На базе **Aiogram** реализована система внешних уведомлений — пользователи могут получать алерты о важных событиях из чата прямо в Telegram.
5.  **Интеграция с OpenAI:** Архитектура подготовлена к работе с нейросетями (генерация ответов, авто-модерация контента или умные подсказки в чате).
6.  **Production-ready Docker Compose:** Стек полностью оркестрован. Nginx настроен на корректное проксирование веб-сокетов с передачей заголовков `Upgrade` и `Connection` на ASGI-сервер Daphne.

## 📦 Развертывание и первый запуск (Local Production)

1. Клонируйте репозиторий и создайте файл `.env` на основе `.env.example`.
2. Запустите весь стек одной командой:

   ```bash
   docker compose up --build
   ```
3. Выполните миграции (происходит автоматически при старте контейнера) и создайте суперпользователя:

   ```bash
   docker compose exec django python manage.py createsuperuser
   ```
   
4. Мессенджер будет доступен по адресу: `http://localhost/`
5. Панель управления очередями RabbitMQ доступна по адресу: `http://localhost:15672/`

### 💬 Как начать тестирование чата:
1. Перейдите в админ-панель по адресу `http://localhost/admin/` под учетной записью созданного суперпользователя.
2. Создайте тестовых пользователей.
3. Перейдите в раздел **ChatRooms** и создайте новый чат, вручную выбрав из списка пользователей, которые должны в него войти.
4. Авторизуйтесь на главной странице под разными пользователями в разных вкладках браузера для тестирования real-time переписки.
